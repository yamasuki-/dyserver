## 概要

作りたいものはWebAPIの呼び出しを検証するためのダミーサーバである

## 要求

- ローカルで動作すること
- AWSで動作すること
- 設定のWebページを持つこと
- 設定のWebページで設定した内容を保存できること
- 設定のWebページで設定した内容を読み込むことができること
- 設定のWebページで設定した内容をリセットできること
- 設定のWebページで設定した内容をエクスポートできること
- 設定のWebページで設定した内容をインポートできること
- エンドポイントパスを動的に複数持てること
- エンドポイントパスごとにGET/POST/PUT/DELETEを有効/無効にできること
- エンドポイントパスごとにGET/POST/PUT/DELETEのレスポンスを動的に設定できること
- エンドポイントパスごとにGET/POST/PUT/DELETEのレスポンスを返さないこともできること
- エンドポイントパスごとにGET/POST/PUT/DELETEのレスポンスのステータスコードを動的に設定できること
- エンドポイントパスごとにGET/POST/PUT/DELETEのレスポンスのヘッダーを動的に設定できること
- エンドポイントパスごとにGET/POST/PUT/DELETEのレスポンスのボディを動的に設定できること
- エンドポイントパスごとにGET/POST/PUT/DELETEのレスポンスのステータス、ヘッダー、ボディをセットとしてレスポンスセットとして保持できること
- エンドポイントパスごとにGET/POST/PUT/DELETEのレスポンスセットを複数持てること
- エンドポイントパスごとにGET/POST/PUT/DELETEのレスポンスセットを動的に切り替えられること
- エンドポイントパスごとにGET/POST/PUT/DELETEの複数あるレスポンスセットをランダムで応答できるようにできること
- エンドポイントパスごとにGET/POST/PUT/DELETEの複数あるレスポンスセットをリクエストボディの特定のキーの値によって応答できるようにできること
- エンドポイントパスごとにGET/POST/PUT/DELETEの複数あるレスポンスセットをリクエストヘッダーの特定のキーの値によって応答できるようにできること
- エンドポイントパスごとにGET/POST/PUT/DELETEの複数あるレスポンスセットをリクエストクエリの特定のキーの値によって応答できるようにできること
- エンドポイントパスごとに、受信したリクエストの内容を保存できること
- エンドポイントパスごとに、受信したリクエストの内容を保存した内容を設定のWebページで確認できること
- 受信したリクエストの内容を保存した内容を設定確認する場合、エンドポイントでフィルタできること
- 受信したリクエストの内容を保存した内容を設定確認する場合、リクエストボディの特定のキーの値でフィルタできること
- 受信したリクエストの内容を保存した内容を設定確認する場合、リクエストヘッダーの特定のキーの値でフィルタできること
- 受信したリクエストの内容を保存した内容を設定確認する場合、リクエストクエリの特定のキーの値でフィルタできること    
- エンドポイントパスごとに、受信したリクエストの内容を保存した内容を設定のWebページで削除できること
- エンドポイントパスごとに、受信したリクエストの内容を保存した内容を設定のWebページで一括削除できること
- エンドポイントパスごとに、受信したリクエストの内容を保存した内容を設定のWebページで検索できること
- エンドポイントパスごとに、受信したリクエストの内容を保存した内容を設定のWebページで並び替えできること
- エンドポイントパスごとに、受信したリクエストの内容を保存した内容を設定のWebページでページングできること
- エンドポイントパスごとに、受信したリクエストの内容を保存した内容を設定のWebページでCSVで出力できること
- エンドポイントパスごとに、受信したリクエストの内容を保存した内容を設定のWebページでJSONで出力できること
- エンドポイントパスごとに、受信したリクエストの内容を保存した内容を設定のWebページでExcelで出力できること
- エンドポイントパスごとに、応答したレスポンス内容を保存できること
- エンドポイントパスごとに、応答したレスポンス内容を設定のWebページで参照できること
- エンドポイントパスごとに、応答したレスポンス内容を設定のWebページでCSVで出力できること
- エンドポイントパスごとに、応答したレスポンス内容を設定のWebページでJSONで出力できること
- エンドポイントパスごとに、応答したレスポンス内容を設定のWebページでExcelで出力できること
- 全エンドポイントに対して、受信したリクエストの内容を保存する・しないを切り替えられること
- 全エンドポイントに対して、受信したリクエストの内容を保存する・しないを切り替えても過去の保存内容は削除しないこと
